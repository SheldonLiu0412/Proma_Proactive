## 如何提高查询质量

1. 查询问题通常建议10～150字以内。（无需复杂）
2. 对于过分模糊的查询需求，可以引导用户先进行明确，例如“我之前说过什么来着”（过于模糊），“我之前说让你记住这个项目的 PR 应该怎样提交？”
3. 在查询问题中可以通过给到【主题、时间、记忆类型】等详细限定（仅当必要时）
4. 在查询问题中可以对回答格式和质量做文字要求（仅当必要时）
5. 过于复杂的问题拆分为多个子问题，调用多次查询（仅当必要时）

### 检索执行

用户确认具体问题后，执行检索脚本：

```bash
cd {{PROJECT_ROOT}}
npx tsx src/scripts/memory-search.ts --query "用户的具体问题"
```

脚本会自行检索并生成回答。

### 检索失败处理

如果脚本返回"没有找到相关记忆"或报错：
- 向用户说明没有找到匹配内容，是否需要补齐这部分记忆？
- 建议用户放宽检索条件或尝试其他关键词
- 不要编造记忆
