## 引导用户规范检索问题

当用户表达检索意图时，先引导他们把需求澄清成**具体的检索问题**。好的检索问题应该包含：

| 维度 | 说明 | 示例 |
|------|------|------|
| **主题** | 具体要找什么信息 | "我常用的模型"、"我的工作流程"、"我对某功能的反馈" |
| **时间范围**（可选） | 大概什么时候的 | "最近一周"、"上个月"、"初始化时" |
| **信息类型**（可选） | 哪类记忆 | "画像里的"、"日记里提到的"、"纠偏记录" |

### 示例对话

**模糊请求**：
> "我之前说过什么来着"

**你的引导**：
> "你想找哪方面的记忆呢？可以具体一点，比如：
> - 关于工作习惯或偏好的
> - 关于某个项目或技术选择的
> - 关于你对我的反馈或纠正的
> - 或者是某个具体时间点提到的"

**规范后的问题**：
> "我之前让你记住我喜欢用什么大模型，现在告诉我"

### 检索执行

用户确认具体问题后，执行检索脚本：

```bash
cd {{PROJECT_ROOT}}
npx tsx src/scripts/memory-search.ts --query "用户的具体问题"
```

脚本内部会：
1. 分析检索范围（哪些文件、什么时间范围）
2. 精准读取相关记忆内容
3. 基于内容生成回答

**直接把脚本的输出返回给用户**，不要自行补充或修改答案。

### 检索失败处理

如果脚本返回"没有找到相关记忆"或报错：
- 向用户说明没有找到匹配内容
- 建议用户放宽检索条件或尝试其他关键词
- 不要编造记忆
