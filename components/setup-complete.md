## 配置完成

满足以下条件后，才算配置完成：

1. `{{PROJECT_ROOT}}/config/memory-instance.local.json` 已存在且内容完整
2. `{{PROJECT_ROOT}}/skills/` 下已生成最新的 `SKILL.md`
3. `~/.proma/agent-workspaces/<slug>/skills/` 下已同步最新 skills

若任一步失败，明确说明失败点与下一步建议，不要假装成功。

---

配置完成后，按以下步骤告知用户后续的完整使用方式。逐步告知，不要一次性把所有内容全部输出，每完成一个环节的说明后等用户确认再继续。

### 下一步告知顺序

**第一步：告知记忆初始化**

向用户说明：

> 配置完成。接下来需要做一次**首次记忆初始化**，这一步会扫描你在 Proma 里的全部历史对话，建立你的用户画像和偏好记录。这个过程可能需要十几分钟，取决于你的历史会话数量。
>
> **操作步骤：**
> 1. 打开 Proma，切换到刚才绑定的 Memory 工作区
> 2. 新建一个会话，模型选 **claude-sonnet-4-6**
> 3. 把下方这段 prompt 完整粘贴发送：
>
> ```
> 任务：执行 memory-init 流程，从全部历史会话初始化记忆。
> 状态：任务监督评估已介入。
> 注意！严格遵守SKILL指导工作，不允许自行探索历史会话目录，上下文和原始存储！
> ```
>
> 发送后 Agent 会自动开始运行，中途不需要干预，等它说"初始化完成"就好。
>
> 完成了告诉我，我来继续说明每日使用方式。

等用户回复后继续说明第二步。

---

**第二步：告知每日记忆整合**

用户确认初始化完成后，向用户说明：

> **每日记忆整合**是这个系统的日常使用方式——建议每天结束工作前跑一次，把今天的对话内容提炼进长期记忆。
>
> **操作步骤：**
> 1. 打开 Proma，切换到 Memory 工作区
> 2. 新建一个会话，模型选 **claude-sonnet-4-6**
> 3. 把下方这段 prompt 完整粘贴发送：
>
> ```
> 任务：执行 memory-daily 流程，整合今日会话记忆。
> 状态：任务监督评估已介入。
> 注意！严格遵守SKILL指导工作，不允许自行探索历史会话目录，上下文和原始存储！
> ```
>
> 同样等 Agent 自动跑完即可，通常 3–5 分钟。
>
> 还有自动化运行的方式，要了解吗？

等用户回复后继续说明第三步（可选）。

---

**第三步（可选）：自动化运行**

若用户想了解自动化运行，向用户说明：

> **自动化方式**是通过 `memory-runner.mjs` 脚本，设置一个 cron 任务，让系统在每天固定时间自动跑 memory-daily，不需要你手动操作。
>
> 如果你想配置自动运行，告诉我你希望每天几点跑，我来帮你配置 cron。
