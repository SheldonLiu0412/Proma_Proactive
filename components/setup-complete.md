## 检查环节

满足以下条件后，才算配置完成：

1. `{{PROJECT_ROOT}}/config/memory-instance.local.json` 已存在且内容完整
2. `{{PROJECT_ROOT}}/skills/` 下已生成最新的 `SKILL.md`
3. `$HOME/.proma/agent-workspaces/<slug>/skills/` 下已同步核心 skills（memory-daily、memory-edit、memory-init、memory-init-review、memory-search），**不应有 memory-setup**

- 检查出现问题则回退前置步骤检查；若检查无误，则直接告知用户后续的完整使用方式。

---

### 一次性告知 Proma Memory 组件完整使用方式

**第一步：告知记忆初始化**

向用户说明：
配置完成，配置位于...，使用需要通过在专属工作区手动发送任务指令。接下来需要做一次**首次记忆初始化**，这一步会扫描你在 Proma 里的近期历史对话，建立你的用户画像和偏好记录等初始记忆。

向用户推荐：
- 该工作区仅用于 Memory 运行，可以关闭除 Memory 预置以外的其它Skill以节约 Token 消耗；
- 模型选 **claude-sonnet-4-6** 及以上，保证一定的初始化质量，对于后续记忆迭代起奠基作用；

操作步骤：
1. 切换到刚才绑定的 Memory 工作区
2. 新建一个会话，模型选 **claude-sonnet-4-6**
3. 把下方这段 prompt 完整粘贴发送：

```
任务：执行 memory-init 流程，从全部历史会话初始化记忆。
状态：任务监督评估已介入。
注意，请严格遵守SKILL指导工作，不允许自行探索历史会话目录，上下文和原始存储。
```

发送后 Agent 会自动开始运行，等待其初始化完成即可，若途中意外中断，可以输入继续直到任务全部完成。

---

**第二步：告知每日记忆整合**

**每日记忆整合**是这个系统的日常使用方式——建议每天结束工作前跑一次，把今天的对话内容提炼进长期记忆；执行会比初始化消耗少很多，也更快。

操作步骤：
1. 切换到 Memory 工作区
2. 新建一个会话，模型选 **claude-sonnet-4-6**
3. 把下方这段 prompt 完整粘贴发送：

```
任务：执行 memory-daily 流程，整合今日会话记忆。
状态：任务监督评估已介入。
注意！严格遵守SKILL指导工作，不允许自行探索历史会话目录，上下文和原始存储！
```

**第三步：告知日常使用方法**

已经自动向 Memory 专用工作区部署了记忆查询和记忆修改 Skills，可以自行添加至其它所需工作区，通过 /skill 的方式主动查询记忆或专程修改记忆，查询和修改操作依赖 Anthropic 兼容格式 LLM API 配置，配置方式为：

修改 {{PROJECT_ROOT}}/config/llm-config.json，填入你的 API 地址、密钥和模型配置（仅替换value）。

需要的话在当前会话内随时告知，我来帮助你完成配置。

**第四步：告知注意事项**
- 当前是一个Demo版本，使用方法有些笨，不同用户间效果差异大，仅用于测试记忆提炼效果；后续将通过 Proma 插件市场方式提供更完整更丰富的集成功能，例如后台执行、自适应启动/手动调用的记忆检索工具，记忆管理面板等等，详见 https://github.com/SheldonLiu0412/Proma_Proactive 后续开发计划；
- 当前流程已完毕，memory-setup 仅用于首次安装引导，安装完成后即可结束当前会话
