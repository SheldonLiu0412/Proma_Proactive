## 引导用户规范操作需求

当用户表达修改意图时，帮助用户转化为好的规范操作需求；对于用户模糊的需求要引导用户进行澄清；好的操作需求应该包含：

| 维度 | 说明 | 示例 |
|------|------|------|
| **操作类型** | 增/删/改 | "添加"、"更新"、"删除" |
| **目标** | 修改哪里 | "画像里的技术偏好"、"某条纠偏记录"、"某个 SOP" |
| **内容** | 改成什么 | 具体的新内容描述 |

> **补充提示**：对于纠偏记录（correction），如果用户能区分「简短摘要」和「详细背景说明」，鼓励其分别描述；脚本提示词会自动拆分处理。

### 列出记忆目录结构（仅供读取）

在操作前，你可以先查看当前记忆的完整目录结构，以便用户需要读取记忆原文的时候使用：

```bash
cd {{PROJECT_ROOT}}
npx tsx src/scripts/memory-edit.ts --list
```

该命令会输出 `.memory/` 目录的完整树形结构，包含所有文件和大小。

### 操作执行流程

在执行修改前，**应先向用户复述具体最终修改需求并确认**，用户确认后，执行操作脚本：

```bash
cd {{PROJECT_ROOT}}
npx tsx src/scripts/memory-edit.ts --instruction "用户的具体操作需求"
```

脚本内部会：分析操作需求 → 输出结构化操作计划 → 映射到具体操作 → 返回操作结果

**把脚本的输出返回给用户**。

### Memory 系统支持修改的记忆类型

| 类型 | 说明 | 执行方式 | 示例需求 |
|------|------|----------|----------|
| **画像更新** | 修改 profile.md | 脚本返回具体指导，**你使用 Edit 工具手动编辑** | "记住我的工作领域是 AI Infra，我最喜欢吃的水果是..." |
| **纠偏记录** | 添加/编辑/删除 correction | 脚本直接修改 | "记录：summary 是「翻译返回双语」，detail 是「每次翻译必须同时输出中英文对照」" |
| **SOP 删除** | 删除 SOP 候选 | 脚本直接删除 | "删除 sop_001 / XXX的那条sop" |

### Profile 编辑指导

当脚本返回 profile 编辑指导时，你会收到：
- profile.md 的完整文件路径
- 用户画像写作规范（视角、结构、风格要求）
- 建议的目标段落和内容

请使用 Edit 工具直接编辑该文件，**不允许重新创建或删除 profile.md**。
